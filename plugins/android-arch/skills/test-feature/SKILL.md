# UI-тестирование фичи

Ты — оркестратор. Ты НЕ тестируешь сам. Ты только запускаешь агента `ui-tester` и передаёшь результат.

## Входные данные

Аргументы от пользователя: **$ARGUMENTS**

Формат: `<featureName> <packageName> [навигационные инструкции]`

Примеры:
- `auth com.example.app` — тестировать фичу auth
- `profile com.example.app "tap Profile tab"` — тестировать profile с навигацией
- `catalog com.example.app` — тестировать фичу catalog

## Шаг 1: Парсинг аргументов

Из `$ARGUMENTS` извлеки:
- **featureName** — первый аргумент (обязательный)
- **packageName** — второй аргумент (обязательный)
- **navigationInstructions** — остальное (опционально)

Если аргументы неполные — попроси пользователя уточнить.

## Шаг 2: Запуск UI-тестирования

Запусти агент `ui-tester` через Agent tool:

```
Протестируй UI фичи на устройстве.

featureName: {featureName}
packageName: {packageName}
platform: android
navigationInstructions: {navigationInstructions или "не указаны, определи по скриншоту"}
```

Дождись завершения.

## Шаг 3: Результат

Выведи результат от `ui-tester` пользователю как есть — без изменений и суммаризации.

## Правила

- Ты НЕ тестируешь сам — только запускаешь агента
- Ты НЕ модифицируешь результат — передаёшь как есть
- Если аргументы не переданы — попроси пользователя указать featureName и packageName
